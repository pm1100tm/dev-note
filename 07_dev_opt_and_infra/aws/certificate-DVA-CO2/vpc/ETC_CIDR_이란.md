# 🚀 CIDR 이란?

🌐 CIDR (Classless Inter-Domain Routing)이란?

- CIDR = IP 주소 범위를 표현하는 표기법.
- IP주소/Prefix 형태로 네트워크의 크기(사용 가능한 IP 개수)를 결정한다.

```shell
10.0.0.0/16
192.168.1.0/24
172.31.0.0/20
```

<br>

## 🧠 2️⃣ Prefix(슬래시 뒤 숫자)는 무엇인가?

- /16, /24 와 같은 숫자는 네트워크 비트 수를 의미한다.

- ➡️ 숫자가 커질수록 네트워크는 작아지고, 호스트(사용 가능한 IP)는 줄어든다.
- ➡️ 숫자가 작을수록 네트워크는 커지고, 호스트는 많아진다.

<br>

## 📊 3️⃣ Prefix에 따른 IP 개수

```shell
| CIDR | 서브넷 마스크 | 사용 가능한 IP 개수 |
|------|---------------|----------------------|
| /8   | 255.0.0.0     | 16,777,216개 |
| /16  | 255.255.0.0   | 65,536개 |
| /24  | 255.255.255.0 | 256개 |
| /28  | 255.255.255.240 | 16개 |
```

IPv4 주소는 총 32비트임.

- /<prefix> 에서 / 뒤 숫자 = 네트워크 부분(bit) 개수
- 나머지 비트 = 호스트 부분(bit) → 이걸로 실제 IP 개수를 만듦

<br>

예를 들어:

🧮 /16 일 때

- 네트워크 비트: 16비트
- 호스트 비트: 32 - 16 = 16비트
- 가능한 IP 개수: 2¹⁶ = 65,536개

🧮 /24 일 때

- 네트워크 비트: 24비트
- 호스트 비트: 32 - 24 = 8비트
- 가능한 IP 개수: 2⁸ = 256개

<br>

## 🏗 4️⃣ AWS에서 CIDR이 사용되는 곳

CIDR은 AWS 네트워킹의 거의 모든 영역에서 등장한다.

```shell
| 서비스 | 예시 |
|--------|--------|
| **VPC** | 10.0.0.0/16 |
| **Subnet** | 10.0.1.0/24, 10.0.2.0/24 |
| **Security Group** | 0.0.0.0/0, 10.0.0.0/16 |
| **Route Table** | 0.0.0.0/0 (인터넷 게이트웨이) |
| **VPC Peering** | CIDR 겹치면 불가 |
| **VPN** | On-Premise CIDR ↔ AWS CIDR 매핑 |
```

- 즉, CIDR은 AWS 네트워크 전체를 정의하는 “주소 공간 설계도”와 같다.

<br>

## 🔍 5️⃣ 실제 예시로 이해하기

예를 들어:

```shell
VPC CIDR: 10.0.0.0/16
```

이 의미는?

- IP 범위: 10.0.0.0 ~ 10.0.255.255
- 사용 가능한 주소 약 65,000개
- 이 안에서 Subnet을 이렇게 나눔:

```shell
Public Subnet A: 10.0.1.0/24
Public Subnet B: 10.0.2.0/24
Private Subnet A: 10.0.10.0/24
Private Subnet B: 10.0.20.0/24
```

- ➡️ 즉, CIDR은 네트워크 크기를 표현하고 subnet 설계의 기초가 된다.

<br>

## 요약

### CIDR란?

- Classless Inter-Domain Routing의 약자
- 네트워크 주소 범위를 나타내는 표기법
- 형식: `IP주소/Prefix`
- Prefix 숫자가 클수록 네트워크는 작아지고 IP 개수는 적어짐

### Prefix별 IP 개수 예

- `/16` → 약 65,536개
- `/24` → 256개
- `/28` → 16개

### AWS 사용 예시

- VPC CIDR: `10.0.0.0/16`
- Subnet CIDR: `10.0.1.0/24`
- Security Group: `0.0.0.0/0` (전체)
- Routing: `0.0.0.0/0 → IGW`

- CIDR = 네트워크 주소 범위를 나타내는 표기법이며,
  AWS에서 VPC/Subnet/Security Group 전체 구조의 근간이 되는 핵심 개념이다.
