# 🚀 Lambda Pricing

💰 AWS Lambda Pricing — 제대로 이해하기

## 1️⃣ Lambda 과금은 딱 2가지다

- ① 요청 수 (Requests)
- ② 실행 시간 × 메모리 (GB-seconds)

## 2️⃣ 요청 수 과금 (Pay per call)

| 항목      | 값                      |
| --------- | ----------------------- |
| Free Tier | 1,000,000 requests / 월 |
| 초과 요금 | $0.20 / 1M requests     |
| 단가      | $0.0000002 / request    |

👉 요청이 100만 이하면 요청 비용 = 0원

## 3️⃣ 실행 시간 과금 (Pay per duration)

핵심 공식

```text
GB-seconds = 실행 시간(초) × 메모리(GB)
```

- 1ms 단위로 과금
- 실행 시간은 올림 처리

## 4️⃣ Free Tier 정확한 의미 (시험 단골)

Free Tier 제공량

- 400,000 GB-seconds / 월

- 이걸 메모리 크기에 따라 다시 해석해야 한다.
- 메모리별 Free Tier 환산

| 메모리            | Free 실행 시간 |
| ----------------- | -------------- |
| 1 GB              | 400,000초      |
| 512 MB (0.5 GB)   | 800,000초      |
| 128 MB (0.125 GB) | 3,200,000초    |

👉 메모리 작을수록 무료 실행 시간 증가

## 5️⃣ Free Tier 초과 시 비용

| 항목      | 값                         |
| --------- | -------------------------- |
| 단가      | $1.00 / 600,000 GB-seconds |
| 1 GB 기준 | $0.000001667 / 초          |

<br>

## 6️⃣ 🔢 실전 계산 예제 (시험 스타일)

### ✅ 예제 1

- 512MB Lambda가 1초 실행, 1,000,000번 호출

```shell
# 요청 비용
1,000,000 requests → FREE

# 실행 시간
메모리 = 0.5 GB
실행 시간 = 1초
총 실행 시간 = 1,000,000초

GB-seconds = 1,000,000 × 0.5 = 500,000 GB-seconds

- Free Tier: 400,000 GB-seconds
- 초과: 100,000 GB-seconds

# 실행 비용
비용 = 100,000 / 600,000 × $1.00
     ≈ $0.17

👉 총 비용 = $0.17
```

### ✅ 예제 2

- 128MB Lambda, 200ms 실행, 2,000,000번 호출

```shell
# 요청 비용
2,000,000 - 1,000,000 = 1,000,000 초과
>> $0.20


# 실행 시간
200ms = 0.2 초

GB-seconds = 2,000,000 × 0.2 × 0.125
           = 50,000 GB-seconds

>> 👉 Free Tier 안에서 끝 → 실행 비용 0

👉 총 비용 = $0.20
```

## 7️⃣ 중요한 성능 vs 비용 포인트 (실무 + 시험)

❗ RAM 늘리면 비용만 늘어날까?

```shell
RAM ↑ → CPU ↑ → 실행 시간 ↓

# 👉 총 GB-seconds가 오히려 줄 수 있음
```

예시

```shell
512MB → 2초 실행
1024MB → 1초 실행
```

- 👉 비용 동일
- 👉 성능은 2배

## 8️⃣ Lambda가 “보통 매우 싼” 이유

| 이유         | 설명 |
| ------------ | ---- |
| Idle 비용    | 0    |
| 서버 유지비  | 0    |
| Auto Scaling | 무료 |
| Free Tier    | 큼   |

👉 트래픽이 들쭉날쭉할수록 Lambda가 압승
