# 실행 계획 분석 체크리스트(MySQL)

실행 계획을 볼 때마다 이 순서로 점검하면 병목을 빠르게 찾을 수 있다.

- ✅ 1. rows 예측 vs 실제 괴리가 10배 이상인가? → ANALYZE TABLE 검토
- ✅ 2. Table scan이 발생하는가? → 인덱스 누락 또는 통계 오류
- ✅ 3. loops가 큰 노드가 있는가? → actual time × loops = 실제 비용
- ✅ 4. Materialize(CTE 물리화)가 불필요한가? → CTE 인라인 처리 검토
- ✅ 5. 조인 결과에 실제로 사용되지 않는 테이블이 있는가? → 불필요한 조인 제거
- ✅ 6. 전체 actual time 중 어느 노드가 가장 큰 비중을 차지하는가? → 그곳이 병목
