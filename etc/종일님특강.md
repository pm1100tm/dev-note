AI 가 프로그램을 인식하는 방법은 문자열이 아니다.
커서같은 것은 추상 구문 트리로 변경해서 인식한다.

> AST 예제라고 검색해볼 것
> AI LSP 를 켜면, 각자의 알고리즘을 가지고 프로그램을 분석해서 읽음
> LLM 에 스트링을 그냥 주면.. 잘못함

- PHP Parser 를 만들어야 한다
- 파싱을 한 다음에 node 와 edge 로 만든다
- 코드를 구문화해서 그래프 DB로 만드는게 중요하다 - 대규모 프로젝트에서
  - 클래스와 메소드의 관계를 분석하는 것이 중요함
  - 시퀀셜 플로우, 콜 플로우
  - 이걸 가지고 LLM에 요청을 함..
  - 코드를 분석할 때 스트링을 분석하면 안된다
  - 그래프 DB가 있다는 것을 알아야 한다 -> 분석해서 그래프 DB에 넣어눠야 함
  - 코드를 분석하는 것은 entry point 부터 분석하는 것임

LLM은 확률 모델 -> 비결정론적

- 컨텍스트가 쥐어지면, 그 다음에 나올 답변 next token, next token 을 확률로 결정하는 것
- 컨텍스트를 좁게해야 하고, 정규화 해야 하고
  - 가장 좋은 것은 확률을 정해주는 것

AI Agent

- LLM + 툴 + LLM이 툴을 어떻게 써야 하는지 플랜을 가지고 있는 것이 AI Agent
  - 어떻게 효과적으로 툴을 써야 하는지 알고리즘을 가지고 있는 것을 AI Agent 라고 함
  - 어떠한 단계를 거쳐서 하는가를 생각해야 한다

- fewshot 퓨샷
