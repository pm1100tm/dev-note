# 텍스트 검색 시나리오

## 1. 개요

```
텍스트 질의 입력 시, 질의 조합(대화 맥락)
→ 질의 분석(의도/옵션/가격/정렬)
→ 임베딩 변환
→ 유사도 검색(Qdrant)
→ 가중 정렬
→ 후처리(재고/임계치)
→ 결과·유사질문·로그 저장의 순서로 처리
```

## 2. 질의 조합

### 2.1 검색 ID(search_id) 처리 규칙

- search_id 제공 시: 직전 질의와 연관성 판단 후, 연관성이 있을 때만 질문을 조합하여 확장
- search_id 미제공 시: 조합 없이 현재 질문만 사용

### 2.2 연관성 판단 기준(예)

- 대명사/수식어가 직전 질의를 참조(예: "그중에", "가격대가 높은")
- LLM 질의 분석 결과에 따라 연관 여부 판단

### 2.3 조합 예시

- 1차: "어머니가 좋아할만한 상품을 찾아줘"
- 2차: "가격대가 높은 상품으로 찾아줘"
- 조합 결과: "어머니가 좋아할만한 가격대가 높은 상품을 찾아줘"

## 3. 질의 분석

질문에서 아래 항목을 추출한다.

- 상품명(question_name): 핵심 제품군(예: "4K TV")
- 상품정보(question_desc): 상품의 기능 및 내용 등 상세 서술
- 리뷰정보(question_review): 긍정 리뷰에서 검색할 키워드(예: "Dolby Atmos", "Black")
- 카테고리(categories): Category3/4와 신뢰도 점수(0~100)
- 상품 옵션(options): 사전 정의 10종 중 해당되는 타입
- 옵션 상세(analyze_options): 옵션별 후보값 리스트(예: color→Black)
- 정렬(sorting / sorting_type): 0~6 코드 및 타입 상수
- 가격(where): 가격 조건 여부/금액/타입(이상·이하·근처)

### 3.1 상품 옵션(10종)

capacity, color, flavor, ingredient, material, pattern, sex, size, type, warranty

### 3.2 정렬 타입(6종)

LG_RECOMMENDED(기본) / PRICE_DESC / PRICE_ASC / RATING_DESC / REVIEW_DESC / NEWEST_DESC

## 4. 임베딩 변환(Embedding)

- 입력: question_name / question_desc / question_review
- 처리: 전처리(불용어 제거·정규화) → 문장 단위 임베딩 → 네임드 벡터 매핑
- 출력: Qdrant 컬렉션의 네임드 벡터(name_vec, desc_vec, review_vec)로 색인/검색 가능 형태로 변환

## 5. 상품 유사도 검색(Scoring)

### 5.1 필터(Pre-Filter)

파트너 ID, 가격대(조건 변환: 이상/이하/근처), 카테고리3/4

### 5.2 유사도 구성(가중 평균)

- 이름 25% + 상세 25% + 리뷰요약 25% + 태그 25%
- 보정: 리뷰 평점 가중치(예: 평점 0~5를 0~1로 정규화 후 가중치 적용)

### 5.3 옵션 정렬

- 옵션 일치 항목 가중 승수 또는 상단 고정

### 5.4 종합 점수

상품 스코어 90% + 신뢰도/인기도 10%(신뢰도/인기도: 평점, 리뷰수, 매출, 긍정 리뷰 비율 등)

- 평점
- 리뷰수
- 매출
- 상품 긍정 리뷰 비율

### 5.5 임계치·컷오프

- 최대 유사도 < 0.4 → 상위 5건만 반환
- 최대 유사도 ≥ 0.4 → 0.4 이상만 필터 후 반환
